# Триде

pnpm monorepo for learning 3D and experimentation with Three.js, R3F, WebGL, WebGPU, game development on the web using Three.js, Blender and other topics related to 3D.

## Role of Claude Code in This Project

Claude Code here is used for **documentation and learning support only**:
generating lesson docs, math explanations, and links indices from completed exercises. It should not write, refactor, or scaffold app code inside `apps/` or `experiments/` unless explicitly asked — app code is written by hand, by design, for practice and learning. Interpreting and documenting the apps and experiments is the most important thing Claude Code does in this monorepo.

## Structure

- `apps/` — standalone apps
- `experiments/` — more experimental apps (things that need to be studied more and their development repeated). Note: `docmaker`/`mathmaker` skills currently only target `apps/` — if something lives in `experiments/` and needs documenting, move/copy it into `apps/` first, or extend the skills later.
- `docs/` — Obsidian vault, plain markdown, maybe a documentation web app in the future
- `packages/` — shared code when it naturally emerges
- `scripts/` — sh scripts used by npm scripts at root level of the monorepo
- `templates/` — reusable file templates (e.g. `README.md`, `MATH.md`) copied into new apps, never edited in place
- `configs/` — TypeScript and other linting/formatting configurations reused across apps and experiments

## Tech

- Apps and experiments use Vite + vanilla Three.js or Vite + R3F depending on the experiment
- TypeScript throughout
- pnpm for everything, no npm or yarn

## Docs

- Notes live in `docs/` as an Obsidian vault of markdown files
- Lesson frontmatter fields: `title`, `topic`, `date`, `tags`, `difficulty`, `app_path`
- Math doc frontmatter fields: `title`, `topic`, `date`, `tags`, `difficulty`, `app_path`
- Lessons organized as `docs/glavno/{topic}/lessons/{lesson_number}_{lesson_name}.md`
- Each lesson topic has an index file at `docs/glavno/{topic}/{topic}.md` with wiki-links to all its lessons
- Math docs organized as `docs/math/{topic}/{concept}.md`
- Math topic index files are not yet auto-generated by `mathmaker` — add this later if the number of math docs per topic grows enough to need one

## Math Explanation Style

When generating math docs, always:
- Use LaTeX: `$inline$` and `$$block$$`
- Write every step on its own line, every `=` on a new line
- Never skip steps or write "it's easy to see that..."
- Build from the user's own attempt at understanding if provided
- Include GeoGebra/Desmos URL with params for geometry/matrix concepts
- Include ShaderToy link for shader-related concepts
- Include 3Blue1Brown link where a relevant video exists

## Link Convention

- All internal links inside `docs/` use Obsidian wiki-links: `[[lesson-name]]`
- Topic index files include a note that links only work in Obsidian
- External links are categorized as: `Docs`, `Examples`, `Tools`, `Articles`, `Videos`, `Courses & Talks`, `Repos`, `Other`
- Unlike user-provided links, LLM-suggested links are marked with `🤖 suggested`
- Outdated link alternatives are flagged in `## Revisit`, never silently replaced

## Conventions

- Each experiment gets its own folder inside the relevant app
- Own variations preferred over reproducing course material exactly
- When something is outdated from course material, note it in the relevant doc
- Never edit `templates/` files directly, always copy first

## Rules

- Never edit files in `templates/` directly — copy into the app folder first
- Never install npm or yarn, only pnpm
- Do not create `packages/` entries speculatively — only when shared code naturally emerges
- `docmaker` must never create or modify files in `docs/math/`
- `mathmaker` must never create or modify files in `docs/glavno/`

## Skills

- `docmaker` — reads a completed app in `apps/{folder-name}` (plus its `README.md` and code comments) and generates a lesson doc in `docs/glavno/{topic}/lessons/`, updating the topic index. Invoke with `/docmaker {folder-name}`.
- `mathmaker` — reads a completed app's `MATH.md` (plus source code) and generates a math/physics/mechanics explanation doc in `docs/math/{topic}/`. Invoke with `/mathmaker {folder-name}`.

## Maintenance

- Once a month, do a manual review or ask Claude Code to scan `docs/` for broken links.
