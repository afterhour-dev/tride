
Evo celog procesa, korak po korak.

## 1. Instalacija

```
/plugin marketplace add anthropics/claude-plugins-official
/plugin install skill-creator@claude-plugins-official
/reload-plugins
```

Ako je marketplace već dodat ranije, samo:

```
/plugin marketplace update claude-plugins-official
/plugin install skill-creator@claude-plugins-official
/reload-plugins
```

Posle `/reload-plugins` skill treba da bude odmah dostupan bez restarta cele sesije.

## 2. Pokretanje

```
/skill-creator
```

ili prirodnim jezikom, npr.: _"napravi mi skill koji X"_, _"evaluiraj moj summarize-changes skill"_, _"poboljšaj moj deploy skill na osnovu ovih test slučajeva"_.

Skill sam prepoznaje **gde si u procesu** — možeš krenuti od nule ili doneti gotov draft i skočiti pravo na eval/iterate deo.

## 3. Ceo ciklus — 4 moda

**Create** — razjasni šta skill treba da radi, napravi draft SKILL.md, i po potrebi test-prompt-ove.

**Eval** — test slučajevi se čuvaju kao prompt/input/expected-behavior trojke u `evals/evals.json` unutar foldera skilla. Svaki test se pokreće u **izolovanom subagentu** (čist kontekst, bez zagađenja između testova), beleži broj tokena i trajanje. Rezultat ide u `grading.json` — pass/fail sa dokazom za svaku tvrdnju (assertion).

**Improve** — na osnovu eval rezultata (kvantitativnih) i tvoje kvalitativne ocene (kroz HTML review viewer), skill-creator prepravlja SKILL.md ciljano — ne prepisuje sve iznova, već hirurški menja slabe delove.

**Benchmark** — agregira pass rate, vreme i tokene za "sa skillom" naspram "bez skilla" u `benchmark.json`, plus **blind A/B poređenje** dve verzije (Comparator agent ne zna koja je stara a koja nova verzija — eliminiše subjektivnost).

## 4. Poseban flow — tuning opisa (trigger problem)

Ako ti se dešava da se skill ne pokreće kad treba, ili se pokreće kad ne treba (npr. skill za YouTube skriptu se aktivira na "napravi mi LinkedIn post"), postoji zaseban skript koji:

- generiše prompte koji **treba** da okinu skill i prompte koji **ne treba**
- meri hit rate na oba seta
- predlaže izmenu `description` polja u frontmatter-u

## 5. Ako ti treba brzo, bez formalnosti

Ako ne želiš celu eval/benchmark mašineriju, možeš mu reći direktno: _"ne treba mi gomila evaluacija, samo hoću da vibe-ujemo"_ — skill-creator je fleksibilan i preskače formalni loop ako to tražiš.

## Za `polygon-lab`

S obzirom da imaš nested `.claude/skills/` po paketu u monorepo-u, prirodan use case bi bio npr:

```
"napravi mi skill koji generiše markdown dokumentaciju lekcije za Three.js poglavlje, 
u istom stilu kao prethodne lekcije u polygon-lab"
```

pa onda `evaluiraj taj skill na 3-4 prošle lekcije kao test slučajeve` da vidiš da li format ostaje konzistentan.

Napomena vezano za tvoj OpenRouter setup: eval mod spawn-uje subagente — proveri da ti subagent slot (trenutno Sonnet/Haiku/subagent na Qwen3-Coder-Next) ne pravi probleme sa izolovanim kontekstom; ako primetiš čudno ponašanje u Eval modu, `/doctor` ti je prvi korak dijagnostike.

#  https://github.com/anthropics/skills

neko klonira ovaj repo i samo cpoy paste skillove u svoj projekat

Da, to je legitiman i zvaničan repo — `anthropics/skills` je Anthropic-ov **zvanični javni repozitorijum** primera skillova, sa preko 149k zvezdica i 17.6k forkova. Tako da ako je neko klonirao repo i samo iskopirao foldere lokalno, radi potpuno legitiman i podržan workflow, samo možda ne najbolji.

## Šta se tačno nalazi u repou

- `skills/` — primeri po kategorijama: Creative & Design, Development & Technical, Enterprise & Communication, i Document skills
- `skills/docx`, `skills/pdf`, `skills/pptx`, `skills/xlsx` — ovo **nisu** samo demo primeri, ovo su isti skillovi koji pokreću Claude-ove ugrađene mogućnosti kreiranja fajlova. Source-available (Apache 2.0 za ostale, ali ovi konkretni su "source-available, not open source" — mogu se gledati ali ne nužno slobodno redistribuirati)
- `spec/` — sama Agent Skills specifikacija
- `template/` — prazan template skill za start

## Dva načina da ih dobiješ — ručno kopiranje vs. plugin marketplace

**Ono što je taj neko uradio** (git clone + kopiranje foldera u `.claude/skills/`) potpuno radi, ali gubiš:

- automatska ažuriranja kad Anthropic doradi skill
- namespace zaštitu (`plugin-name:skill-name`) — ako slučajno napraviš svoj skill istog imena, koliduje

**Preporučen, "zvaničniji" način** je preko plugin marketplace mehanizma koji sam repo eksplicitno podržava:

```
/plugin marketplace add anthropics/skills
```

zatim ili kroz meni (`Browse and install plugins` → `anthropic-agent-skills` → `document-skills` ili `example-skills`), ili direktno:

```
/plugin install document-skills@anthropic-agent-skills
/plugin install example-skills@anthropic-agent-skills
```

Nakon instalacije samo pomeneš skill u promptu, npr. _"koristi PDF skill da izvučeš form polja iz path/to/fajl.pdf"_.

## Praktična razlika za tebe

Ako je osoba o kojoj pričaš samo ručno prekopirala foldere — svaki skill i dalje radi normalno (Claude Code čita bilo koji validan `SKILL.md` bez obzira odakle je stigao), ali gubi update mehanizam. Za `document-skills` (docx/pdf/pptx/xlsx) posebno bi ti preporučio marketplace instalaciju umesto ručnog kopiranja, jer se ti skillovi aktivno održavaju i unapređuju od strane Anthropic-a — a ručna kopija ti zamrzava verziju u trenutku kloniranja.
