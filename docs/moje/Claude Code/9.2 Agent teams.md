## Šta je to

Agent teams je **eksperimentalna** funkcija — koordinacija više Claude Code sesija koje rade zajedno. Jedna sesija je **team lead**: koordinira posao, dodeljuje zadatke, sintetiše rezultate. Svaki **teammate** je puna, nezavisna Claude Code sesija sa sopstvenim kontekst prozorom, i teammates mogu direktno da komuniciraju međusobno — ne samo sa lead-om.

## Ključna razlika od sub-agenata

Ovo je centralna stvar koju treba da uhvatiš:

||Sub-agenti|Agent teams|
|---|---|---|
|Kontekst|Sopstveni, rezultat se vraća pozivaocu|Sopstveni, potpuno nezavisan|
|Komunikacija|Izveštavaju SAMO glavnom agentu|Teammates direktno šalju poruke jedni drugima|
|Koordinacija|Glavni agent upravlja svim|Deljena task lista, sami se koordinišu|
|Trošak tokena|Niži — rezultat se sažima nazad|Viši — svaki teammate je posebna instanca|

Sub-agent je kao izvođač kog pošalješ na poseban zadatak — vratiš mu izveštaj i to je to. Agent teams je kao tim koji raspravlja, izaziva jedni druge, i sam odlučuje ko šta radi.

## Kako se uključuje

Isključeno po defaultu jer je eksperimentalno:

```bash
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
```

ili u `settings.json`. Bez ove promenljive, Claude uopšte ne pravi tim niti predlaže teammates.

Od v2.1.178, više nema posebnog koraka setup-a (ranije si morao da tražiš od Claude-a da napravi i imenuje tim preko `TeamCreate`/`TeamDelete` alata — ti alati više ne postoje). Sad prosto opišeš zadatak i teammates koje želiš prirodnim jezikom, i Claude ih spawn-uje.

## Primer prompta

```
Kreiraj agent team da refaktoriše polygon-lab shader modul. 
Spawn tri teammate-a: jedan za GLSL performanse, jedan za 
Three.js integraciju, jedan za testove. Neka se koordinišu 
kroz deljenu task listu.
```

## Kada koristiti šta

- **Sub-agent** — brz, fokusiran worker koji ti samo vrati rezultat (npr. onaj `glsl-checker` ili `docs-writer` o kojima smo pričali)
- **Agent teams** — kad zadatak zahteva raspravu, više uglova gledanja, ili međusobno proveravanje nalaza (npr. istraživanje sa tri različita ugla, code review gde jedan teammate izaziva nalaze drugog)

## Cena — bitno za tebe

Za tim od 3 teammate-a, potrošnja tokena je otprilike **3-5x veća** nego kod jedne sesije koja isti posao radi sekvencijalno — svaki teammate je posebna Claude instanca koja učitava sopstveni kontekst od nule. S obzirom da paziš na OpenRouter kredite, ovo bih koristio štedljivo — samo za zadatke gde paralelna rasprava stvarno donosi vrednost, ne za rutinski rad koji sub-agent ili čak glavna sesija mogu da odrade sami.

## Ograničenja

Dokumentacija eksplicitno kaže da agent teams ima poznata ograničenja oko **nastavka sesije (resume)**, koordinacije zadataka i gašenja teammates-a — dakle ne oslanjaj se na ovo za produkcioni workflow, tretiraj ga kao work-in-progress feature.
